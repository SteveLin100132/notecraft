理解 Event Loop 是搞懂非同步行為的前提。關鍵在於：每跑完一個 macrotask，引擎會清空整個 microtask 佇列，才繼續下一個 macrotask。

## 執行順序

1. 同步程式碼先在 call stack 上跑完。
2. Promise 的 `.then` / `await` 之後屬於 microtask，優先於 `setTimeout`。
3. 每個 macrotask 之間，microtask 佇列會被完全清空。

```js
console.log('A');
setTimeout(() => console.log('B'));
Promise.resolve().then(() => console.log('C'));
console.log('D');
// 輸出順序：A D C B
```

> microtask 永遠插隊在下一個 macrotask 之前。
