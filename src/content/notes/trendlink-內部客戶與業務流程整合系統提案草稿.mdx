# Trendlink 內部客戶與業務流程整合系統提案

## 背景

客服人員與顧問師在日常工作中，必須針對各自負責的客戶合約進行管理與追蹤。
現行的作業方式為：每當與客戶洽談或更新合約內容時，皆需手動將合約相關資訊維護至 Excel 檔案中。透過 Excel 進行合約資料的登錄、更新與追蹤。

現階段使用 Notion 作為業務管理的過渡期工具，根據 [合約與客戶管理系統企劃](https://www.notion.so/trendlink-workspace/31b43e6601e28023a65fc7d3c41341c8?source=copy_link)，
以 Notion 作為 PoC 的期間，同步搜集使用者需求，並記錄相關規格與流程，作為未來系統開發的參考。

:::info{title="目前進度"}
針對**合約**與**客戶**管理的部分，相關規格已趨於成熟，並可作為未來系統開發的參考。
:::

---

## 痛點

在使用 Notion 進行客戶與合約管理時，面臨以下問題，可能長期使用會出現難以克服的情境：

- :badge[資料分散]{variant="danger" icon="warning"} 客戶與合約資訊分散在不同的 Notion 資料庫中，雖可以透過關聯進行整合，但無法像傳統 RDB 一樣進行複雜查詢與分析。
- :badge[權限設定]{variant="danger" icon="warning"} 目前 Notion 雖可以按照人員顯示特定的合約資料，但沒有辦法做到 Column Level 的權限設定，導致部分敏感資訊無法被保護。
- :badge[欄位驗證]{variant="danger" icon="warning"} Notion 的資料填寫，無法鎖定必填與格式驗證，導致資料品質不佳。
- :badge[效能貧頸]{variant="danger" icon="warning"} Notion 的資料量增加時，查詢與操作效能可能下降，影響使用者體驗。
- :badge[版本迭代]{variant="danger" icon="warning"} Notion 的功能更新與變動，目前在權限的設定上就因為表單權限的變動，導致相關功能無法使用。

## 系統模組 & 服務整合

### 業務流程管理（需具有流程追蹤與交付文件管理能力）

- 對應系統：:badge[BPM]{variant="info" outline}
- 服務選型：:badge[自建系統]{variant="success"}

### 合約管理（需具有電子簽章與法律效力）

- 對應系統：:badge[CLM]{variant="info" outline}
- 服務選型：:badge[點點簽]{variant="tip"} :badge[BizForm]{variant="tip"}

:::note
目前僅有點點簽提供電子簽章服務，BizForm 目前僅提供表單簽核服務，因此在合約管理，選用點點簽最為妥當。
:::

### 客戶管理與商機追蹤

- 對應系統：:badge[CRM]{variant="info" outline}
- 服務選型：:badge[Vital CRM]{variant="tip"}

::::tabs
:::tab{label="Vital CRM 功能特色"}
- 功能可以按權限劃分
- 支援手機 App 方便業務在外也能夠使用
- 商機狀態可以自訂
- 客戶備忘錄可以通過記事填寫，記事也可以設定範本
- 有 Lead Time 與 Cycle Time 的概念，能夠追蹤各商機階段停留幾天
- 具有簽核流程，包含會簽、擇簽、加簽等等
- 知識庫 KM，支援文件管理、自訂欄位、先關文件關聯、可以模糊查詢（包含附件也行）
:::
:::tab{label="Vital CRM 問題釐清"}
   
> ✅ 表示能夠滿足需求、⚠️ 表示能夠滿足需求，但需要結合其他服務才有機會實現或者客制化、❌ 表示無法滿足需求

1. **客戶資料的權限劃分？能客制不同的人只能看到特定的客戶資料嗎？**
   <br/> *Ans：✅ 可按組織或個人設定權限，同時也能夠分享權限（滿足目前多個顧問共同執案的部分）*
2. **客戶資料的欄位能夠自訂嗎？例如說客戶來源的種類（看起來可以用標籤實現）、公司員工人數等。**
   <br/> *Ans：✅ 可以自訂*
3. **商機的狀態可以自訂，那該狀態裡面有辦法提供自訂表單嗎？我們的業務流程有包含派案 -> 諮詢 -> 報價 -> 執案等業務節點，每個業務節點都有要交付的文件（看起來是用人員記事，可以提供範本，但好像不是表單，所以如果要確認能不能進行統計，有看到 BizForm 可能可以透過這個部分實現）**
   <br/> *Ans：⚠️ 參考 Q4，但結論是需要客制*
4. **BizForm 目前綁定對應客戶，那可以綁定對應的商機階段嗎？**
   <br/> *Ans：⚠️ 目前沒辦法，但可以走客制化的流程，或參照 Q5 使用 KM 實現*
5. **另外也有合約管理的需求，目前有哪個服務或模組可以提供？用訂單的方式來實現嗎？**
   <br/> *Ans：⚠️ 使用 KM 知識管理系統，屬於文件管理系統，可以自訂欄位*
6. **KM 與 CRM 的交互，如何知道該客戶底下有哪些文件（合約）？**
   <br/> *Ans：⚠️ CRM 增加網址欄位，加入 KM 文件連結，但需要兩邊同時設定權限*
7. **簽核流程能不能有訪客（渠道人員）？**
   <br/> *Ans：❌ 不可以，需要有系統帳號*
8. **儀表板能否結合 KM？**
   <br/> *Ans：⚠️ 目前僅針對 CRM 本身的資料，如果硬要弄，也可以透過 API，不過透過網址來做資產的關聯，其實偏弱*

## 解決方案

因聯和趨動的業務流程與客戶管理需求，除傳統 CRM 外，甚至跨足 BPM（業務流程管理）、Finance（發票開立與收款）與 CLM（合約生命週期管理），
因此除了自建系統外，也會朝向整合現有平台或服務的方向發展。

### 方案 A

- 主系統：自建系統，整合客戶管理與業務流程管理，作為系統中控
- 客戶管理：自建系統
- 派案管理：整合 BizForm，BizForm 建置表單，交付表單後的資料需與主系統進行資料串接
- 諮詢管理：整合 BizForm，BizForm 建置表單，交付表單後的資料需與主系統進行資料串接
- 報價管理：整合 BizForm，BizForm 建置表單，交付表單後的資料需與主系統進行資料串接
- 合約管理：整合點點簽，電子合約由點點簽管理，相關合約資料需與主系統進行資料同步
- 執案管理：整合 BizForm，BizForm 建置表單，交付表單後的資料需與主系統進行資料串接
- 分析看板：自建系統，提供業務流程的看板視圖，並與 BizForm 進行資料串接，顯示各業務節點的狀態與相關資訊

::::tabs
:::tab{label="優勢"}
- 不用架設相關服務，有人運維
- BizForm 內建簽核流程，不需要搭建（現階段沒有簽核，不代表這是正常流程）
:::
:::tab{label="劣勢"}
- 串接成本要考量，串接 BizForm 需要 Follow 點點簽試用的模式，串接對應 API，需要額外估算所需的資源與時間
- BizForm API 交互需要額外收費
:::
::::

### 方案 B

- 主系統：自建系統，整合客戶管理與業務流程管理，作為系統中控
- 客戶管理：自建系統
- 派案管理：自建系統
- 諮詢管理：自建系統
- 報價管理：自建系統
- 合約管理：整合點點簽，電子合約由點點簽管理，相關合約資料需與主系統進行資料同步
- 執案管理：自建系統
- 分析看板：自建系統，提供業務流程的看板視圖，顯示各業務節點的狀態與相關資訊

::::tabs
:::tab{label="優勢"}
- 高度彈性，可以根據公司的業務流程與需求來設計與實現功能，不受限於第三方服務的限制
- 不需要考慮第三方服務的串接成本與維護成本（使用點點簽的 API 不需要額外收費）
:::
:::tab{label="劣勢"}
- 開發成本較高，需要投入更多的資源與時間來開發與維護系統，並用內部資源來管理 Infrastructure 等基建
:::
::::

### 建議方案

:::success{title="建議採用方案 B"}
方案 A 與方案 B 各有優劣，但整體而言，考量當前內部流程未有簽核流程、BizForm 服務若要串接 API 需另外收費，且業務流程與客戶管理需求尚未完全明確，建議先採用方案 B。
:::

  ## 連線（折線、含協定標籤）

  使用一層 absolute 覆蓋的 SVG 畫 orthogonal polyline（只走水平 / 垂直），每條線中段加白底細邊框的小 pill 標籤標註協定：

  - User → Web UI：HTTPS
  - Web UI → API Server：HTTPS / REST API
  - API Server → Cloud SQL：TCP
  - API Server → Cloud Storage：HTTPS（GCS SDK）
  - API Server ↔ n8n：API + Webhook（資料同步 / 通知）
  - 合約管理（API Server 內的子標籤）↔ 點點簽：API + Webhook
  - [A] 業務流程資料對接 ↔ BizForm：Webhook
