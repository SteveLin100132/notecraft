React 的渲染分為 Render（協調，可中斷）與 Commit（套用到 DOM，不可中斷）兩階段。Fiber 架構讓 Render 階段可以被切片、暫停與恢復，是 Concurrent 特性的基礎。

## 為什麼需要穩定的 key

key 讓 React 在列表 diff 時辨識哪些節點是同一個，避免不必要的卸載與重建。用陣列索引當 key 在重排時會出錯。

```tsx
// 好：穩定且唯一
items.map(item => <Row key={item.id} {...item} />)
// 壞：重排時會錯亂
items.map((item, i) => <Row key={i} {...item} />)
```
