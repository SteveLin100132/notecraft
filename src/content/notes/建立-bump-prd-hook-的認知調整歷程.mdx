- **對話結束時做一次性檢查**:不會像 PostToolUse 那樣每改一個檔案就觸發
- **避免無限迴圈**:不會因為改了 PRD 又再次觸發自己
- **時機自然**:Claude 講完話的瞬間檢查,使用者剛好會看到提醒

### 判斷該不該用 hook 的三個準則

1. **這件事「不能漏」嗎?** → 漏了沒關係的事,寫在 CLAUDE.md 就好
2. **這件事「不需要判斷」嗎?** → 需要語意判斷的,給 AI
3. **這件事「綁定在生命週期事件」嗎?** → 使用者意圖驅動的,給 slash command

三個都 yes 才該用 hook。
